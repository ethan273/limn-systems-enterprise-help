# First Steps by Role

Get started quickly with role-specific guidance tailored to your responsibilities and access level.

## Overview

This guide provides step-by-step onboarding instructions for each user role in Limn Systems Enterprise. Follow the section that matches your role to quickly become productive.

**Available Role Guides:**
- **Sales & Account Managers** - Lead customer relationships and close deals
- **Design Team** - Create and manage design projects
- **Production Managers** - Oversee manufacturing operations
- **Finance & Accounting** - Handle invoicing and financial tracking
- **Quality Control** - Manage inspection workflows
- **Administrators** - Configure system and manage users
- **External Partners** - Customers, designers, factory partners, QC teams

## Sales & Account Managers

**Primary Responsibilities:** Lead management, customer relationships, sales pipeline, order creation

### Week 1: Getting Oriented (Days 1-7)

**Day 1: Login & Profile Setup**
1. Log in via Google Workspace SSO (@limn.us.com email)
2. Complete your profile:
   - Navigate to User Menu (top-right) > Profile
   - Add profile picture
   - Verify contact information
   - Set timezone and preferences
3. Bookmark key pages:
   - CRM Dashboard: `/crm`
   - My Tasks: `/tasks?assignee=me`
   - Orders: `/orders`

**Day 2-3: CRM Familiarization**
1. Review CRM Dashboard:
   - Navigate to `/crm`
   - Understand pipeline metrics (conversion rates, funnel stats)
   - Review recent activities feed
2. Practice navigation:
   - Contacts: `/crm/contacts`
   - Leads: `/crm/leads`
   - Customers: `/crm/customers`
3. Create test contact:
   - Click "New Contact"
   - Fill in sample data
   - Practice converting to lead
   - Practice converting lead to customer

**Day 4-5: Learn Pipeline Management**
1. Understand lifecycle stages:
   - Ambassador (initial contact)
   - Contact (qualified contact)
   - Lead (showing buying intent)
   - Prospect (hot/warm/cold temperature)
   - Client (converted customer)
2. Practice lead progression:
   - Create new lead
   - Update prospect status (cold/warm/hot)
   - Move through pipeline stages
   - Log activities (calls, emails, meetings)
3. Review analytics:
   - Conversion rates
   - Pipeline value
   - Time in stage

**Day 6-7: Order Creation & Management**
1. Create your first order:
   - Navigate to `/orders`
   - Click "New Order"
   - Select customer (or create new)
   - Add line items (products/materials)
   - Set pricing, terms, delivery date
   - Save as quote
2. Convert quote to order:
   - Open saved quote
   - Click "Convert to Order"
   - Verify all details
   - Submit order
3. Track order status:
   - View order dashboard
   - Understand status workflow
   - Practice updating order status

### Week 2: Core Workflows (Days 8-14)

**Daily Workflow:**
1. Check CRM Dashboard for:
   - New leads (respond within 5 minutes for 80% higher conversion)
   - Follow-up reminders
   - Pipeline changes
2. Review My Tasks:
   - Complete assigned tasks
   - Update task status
   - Add notes to tasks
3. Log all customer interactions:
   - Use Activities tab on contact/lead/customer pages
   - Record calls, emails, meetings
   - Include next steps and outcomes

**Weekly Tasks:**
- Monday: Review pipeline, prioritize hot prospects
- Wednesday: Clean up stale leads (move to "lost" if >90 days inactive)
- Friday: Review conversion rates, identify bottlenecks

### Tips for Success

**Lead Qualification:**
- Use BANT framework (Budget, Authority, Need, Timeline)
- Set realistic lead values for accurate forecasting
- Update prospect temperature based on engagement level

**Pipeline Hygiene:**
- Move leads promptly through stages (don't let them sit)
- Log all activities (even quick calls)
- Follow up 6-8 times before marking as "lost"

**Keyboard Shortcuts:**
- `Cmd/Ctrl + K` - Global search (find any contact/lead/order instantly)
- `Cmd/Ctrl + N` - Create new record (context-aware)
- `Cmd/Ctrl + S` - Save form

   - Log communications

**Daily Workflow:**
1. Check Production Dashboard:
   - New work orders from orders module
   - Status updates from factory partners
   - Quality issues requiring attention
2. Review My Tasks:
   - Work order assignments
   - Follow-ups with factories
   - Quality issue resolutions
3. Update work order progress:
   - Mark milestones complete
   - Log production notes
   - Update estimated completion dates

**Weekly Tasks:**
- Monday: Review weekly production schedule, assign new work orders
- Wednesday: Check capacity, rebalance if needed

**Factory Selection:**

- Consider lead times when setting deadlines

**Capacity Planning:**
- Track factory capacity utilization
- Don't overload single factory
- Plan for rush orders with reserve capacity

**Quality Management:**
- Require QC inspections before shipping
- Document all defects and corrective actions
- Track defect rates by factory for performance evaluation

## Finance & Accounting

**Primary Responsibilities:** Invoicing, payment tracking, QuickBooks integration, financial reporting

### Week 1: Getting Oriented (Days 1-7)

**Day 1: Login & Financial System Access**
1. Log in and complete profile
2. Bookmark:
   - Finance Dashboard: `/finance`
   - Invoices: `/finance/invoices`
   - Payments: `/finance/payments`
   - QuickBooks Integration: `/finance/integrations/quickbooks`
3. Verify QuickBooks connection status

**Day 2-3: Finance Dashboard Familiarity**
1. Understand dashboard metrics:
   - Total revenue (monthly/quarterly/annual)
   - Outstanding invoices (amount + count)
   - Payments received (last 30 days)
   - Overdue invoices (aging report)
2. Navigate finance sections:
   - Invoices: `/finance/invoices`
   - Payments: `/finance/payments`
   - Customers: `/crm/customers` (for financial details)
3. Review customer financials:
   - Credit limits
   - Payment terms
   - Outstanding balances
   - Payment history

**Day 4-5: Invoice Management**
1. Understand invoice lifecycle:
   - Draft (created, not sent)
   - Sent (delivered to customer)
   - Partially Paid (payment received, balance remaining)
   - Paid (fully paid)
   - Overdue (past due date, unpaid)
   - Cancelled (voided invoice)
2. Practice invoice creation:
   - Create from order
   - Add line items manually
   - Set payment terms
   - Apply discounts/taxes
   - Preview and send
3. Track payment status:
   - Record payments
   - Apply to invoices
   - Generate receipts
   - Follow up on overdue

**Day 6-7: QuickBooks Integration**
1. Understand sync process:
   - Invoice sync (Limn -> QuickBooks)
   - Customer sync (bidirectional)
   - Payment sync (QuickBooks -> Limn)
   - Chart of accounts mapping
2. Practice sync operations:
   - Manual sync trigger
   - Review sync status
   - Resolve sync errors
3. Run financial reports:
   - Revenue by month
   - Aging report (AR)
   - Customer payment history
   - Invoice status summary

### Week 2: Core Workflows (Days 8-14)

**Daily Workflow:**
1. Check Finance Dashboard:
   - New orders ready for invoicing
   - Payments received
   - Overdue invoices
2. Review My Tasks:
   - Invoice approvals
   - Payment follow-ups
   - QuickBooks sync issues
3. Process invoices:
   - Generate invoices for shipped orders
   - Send invoices to customers
   - Record payments received
   - Follow up on overdue accounts

**Weekly Tasks:**
- Monday: Review outstanding invoices, send reminders
- Wednesday: Reconcile QuickBooks sync
- Friday: Generate weekly financial reports

### Tips for Success

**Invoice Timing:**
- Generate invoices within 24 hours of shipment
- Set payment terms clearly (Net 30, Net 60, etc.)
- Send payment reminders 3 days before due date

**Payment Follow-Up:**
- First reminder: 3 days before due date
- Second reminder: On due date
- Third reminder: 7 days past due
- Escalate: 30 days past due (consider collection)

**QuickBooks Sync:**
- Run manual sync daily (morning)
- Review sync errors immediately
- Keep customer records updated in both systems

## Quality Control

**Primary Responsibilities:** Inspection workflows, defect tracking, quality reporting, factory audits

### Week 1: Getting Oriented (Days 1-7)

**Day 1: Login & QC System Access**
1. Log in and complete profile
2. Bookmark:
   - QC Dashboard: `/qc`
   - Inspections: `/qc/inspections`
   - Defects: `/qc/defects`
   - Reports: `/qc/reports`
3. Review QC standards and checklists

**Day 2-3: QC Dashboard & Inspection Types**
1. Understand dashboard metrics:
   - Pending inspections
   - Pass rate (last 30 days)
   - Defect rate by category
   - Top defect types
2. Learn inspection types:
   - Pre-production (sample approval)
   - In-line (during manufacturing)
   - Final (before shipping)
   - Receiving (incoming materials)
3. Navigate QC sections:
   - Inspections: Create, view, complete
   - Defects: Log, track, resolve
   - Reports: Generate quality reports

**Day 4-5: Inspection Workflow**
1. Understand inspection lifecycle:
   - Scheduled (inspection planned)
   - In Progress (inspection underway)
   - Pass (meets quality standards)
   - Fail (defects found, requires action)
   - Re-inspection (after corrective action)
2. Practice creating inspections:
   - Select work order or product
   - Choose inspection checklist
   - Schedule inspection date
   - Assign inspector
3. Complete sample inspection:
   - Use mobile device or desktop
   - Capture photos (defects, overall quality)
   - Mark checklist items (pass/fail)
   - Add inspector notes
   - Submit inspection report

**Day 6-7: Defect Management & Reporting**
1. Log defects:
   - Link to work order or product
   - Classify defect type (cosmetic, functional, critical)
   - Add photos and descriptions
   - Assign corrective action
2. Track defect resolution:
   - Monitor corrective action progress
   - Schedule re-inspection
   - Verify resolution
   - Close defect record
3. Generate quality reports:
   - Factory performance (defect rate by factory)
   - Defect trends (by type, severity, month)
   - Pass/fail rates over time
   - Cost of poor quality (COPQ)

### Week 2: Core Workflows (Days 8-14)

**Daily Workflow:**
1. Check QC Dashboard:
   - New inspections scheduled
   - Pending corrective actions
   - Overdue re-inspections
2. Review My Tasks:
   - Inspections assigned to you
   - Follow-ups on failed inspections
   - Factory audit schedules
3. Complete inspections:
   - Use mobile app for on-site inspections
   - Upload photos via bulk upload tool
   - Link photos to inspection records
   - Submit reports same day

**Weekly Tasks:**
- Monday: Review weekly inspection schedule
- Wednesday: Follow up on pending corrective actions
- Friday: Generate weekly quality report for production team

### Tips for Success

**Inspection Best Practices:**
- Capture at least 5 photos per inspection (overview + details)
- Document all defects, even minor ones (for trend analysis)
- Complete inspection reports same day (while fresh in memory)

**Defect Classification:**
- **Critical** - Safety issue or functionality failure (requires immediate action)
- **Major** - Noticeable defect affecting aesthetics or performance
- **Minor** - Small imperfection, acceptable if rare

**Factory Communication:**
- Share defect photos immediately with factory
- Request root cause analysis for recurring defects
- Schedule factory audits quarterly

## Administrators

**Primary Responsibilities:** User management, system configuration, permissions, integrations, security

### Week 1: Getting Oriented (Days 1-7)

**Day 1: Admin Access & Responsibilities**
1. Log in with admin privileges
2. Bookmark:
   - Admin Dashboard: `/admin`
   - Users: `/admin/users`
   - Roles & Permissions: `/rbac`
   - System Settings: `/admin/settings`
3. Review admin responsibilities

**Day 2-3: User Management**
1. Understand user types:
   - Internal employees (@limn.us.com OAuth)
   - External users (magic link authentication)
   - Portal users (customer, designer, factory, QC)
2. Practice user operations:
   - Create new user
   - Assign roles
   - Grant/revoke permissions
   - Reset passwords
   - Enable/disable MFA
3. Learn admin allowlist:
   - Auto-provisioning for @limn.us.com employees
   - Manual admin promotion
   - Super admin requirements (MFA mandatory)

**Day 4-5: Roles & Permissions (RBAC)**
1. Understand role system:
   - Pre-defined roles (admin, sales, design, production, etc.)
   - Custom roles (create as needed)
   - Role templates (reusable permission sets)
2. Permission management:
   - Use Inline Permission Editor
   - Grant/revoke individual permissions
   - Customer-scoped permissions (multi-tenant)
   - Permission comparison view (role vs user vs effective)
3. Practice permission scenarios:
   - Temporary access (grant with expiration)
   - Explicit denial (restrict specific actions)
   - Customer-scoped access (limit to specific customer data)

**Day 6-7: System Configuration**
1. Portal access management:
   - Grant portal access to partners
   - Configure allowed modules
   - Revoke access (immediate session invalidation)
   - Monitor portal activity logs
2. Integration settings:
   - QuickBooks connection status
   - SEKO WMS API configuration
   - Resend email integration
   - Supabase storage buckets
3. Security settings:
   - MFA enforcement policies
   - Session timeout settings
   - IP restrictions (if needed)
   - Admin action audit logs

### Week 2: Core Workflows (Days 8-14)

**Daily Workflow:**
1. Check Admin Dashboard:
   - New user registrations
   - Failed login attempts
   - Permission change requests
   - System alerts
2. Review My Tasks:
   - User access requests
   - Role assignment approvals
   - Integration issues
3. Monitor system health:
   - Check error logs
   - Review performance metrics
   - Verify backup status

**Weekly Tasks:**
- Monday: Review new user access requests
- Wednesday: Audit permission changes (admin action logs)
- Friday: Generate weekly system usage report

### Tips for Success

**User Onboarding:**
- Create role-specific onboarding checklists
- Assign permissions using role templates (faster than manual)
- Schedule follow-up after 1 week to answer questions

**Permission Management:**
- Use customer-scoped permissions for multi-tenant isolation
- Document permission rationale in "reason" field (audit trail)
- Review user permissions quarterly (remove unused access)

**Security:**
- Enforce MFA for all admin and super_admin users
- Review admin action logs weekly (detect unauthorized changes)
- Rotate API keys quarterly
